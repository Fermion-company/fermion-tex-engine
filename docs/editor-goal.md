# Editor Goal Notes

## 最終ゴール

TeX コマンドを直接知らないユーザーでも、論文・資料を編集し、きれいな PDF まで出力できるエディタにする。

ユーザーに見せる中心体験は TeX ソースではなく、用途別の編集画面、AI 補助、プレビュー、参照管理、作図ツールで構成する。

## 到達ライン

- きれいな PDF を生成できる段階まで持っていく。
- AI を使った資料作成・編集支援も、この段階までには実装する。
- コードそのものはメニューから開いて編集する場合に限って表示する。
- 通常ユーザーには TeX コードを見せない。

## UI 方針

画面はおおむね次の構成を想定する。

- 左: 文書構造・メニュー・ツール一覧。
- 中央: 編集対象の本文または用途別編集画面。
- 右: プレビュー、または補助パネル。

用途ごとに画面を分ける。

- カスタマイズ用
- 編集用
- 構成用
- 参照管理用
- 作図用

カスタマイズ用・編集用は、ほぼ AI とチャットするだけの画面を想定する。実質的な有料機能として扱う可能性がある。

メニューで切り替えれるように。

## TeX 非表示編集

ユーザーが直接 TeX コマンドを知らなくても編集できる状態を目指す。

例:

```tex
\section{Live LuaLaTeX}
\label{sec:live}
```

この場合、ユーザーには `Live LuaLaTeX` や `sec:live` のような引数の中身だけを見せる。

## AI ツール

期限メモ: 8 月末まで。

AI ツールは、文章・構成・TeX 意匠設計に対して、必要に応じて選択式で必要分だけを入れられるようにする。

`minimum TeX` の支援は、後でネイティブアプリ化する段階に回す。

想定機能:

- 文章作成支援
- TeX 構造の補助
- 必要なコマンド・環境の選択
- 内容に応じた差し込み
- ユーザーが TeX を知らなくても編集できる補助
- ユーザー持ち込みの `.sty` を AI と一緒に現在の文書へ組み込む対話機能
- AI と一緒に独自マクロを設計し、プリアンブルへ追加する機能
- AI と一緒に独自の `tcolorbox` 環境を設計する機能
- AI と一緒に柱・ノンブル・章見出しなどの意匠を設計する機能

## コードメニュー

コードメニューは、TeX ソースへ逃がすためだけの画面ではなく、プロジェクトファイルを管理する画面として扱う。

想定機能:

- `.sty` / `.tex` / `.cls` / `.bib` のファイル選択と保存
- 持ち込み `.sty` の `\usepackage` 追加
- 部・章ごとの `.tex` 分割ファイル作成
- `\input` / `\include` 参照の追加
- 現在の文書が参照している外部ファイル一覧
- 大型文書では、部・章・付録などをファイル分割しても UI 側で迷子にならないようにする

## 設定メニューのブラッシュアップ

- 日付は `\today` をそのまま見せず、「今日の日付」のような自然な表記で扱う。
- パッケージは直接入力だけでなく、「数式」「表」「図と画像」「参照リンク」「作図」「意匠」「コード掲載」など用途別に選べるようにする。
- 文書クラスは和文標準の `jlreq` を基本にしつつ、`ltjsarticle` / `ltjsreport` / `ltjsbook` も選択可能にする。
- TeX を知らないユーザーには用途や自然な表記を見せ、上級者にはパッケージやクラスを直接調整できる余地を残す。

## 参照管理

参照管理画面では、次の項目を扱う。

- `\ref`
- `\label`
- 章
- 節
- 目次
- 図表

参照・ラベル・目次などは、UI 側で管理できるようにする。

## 作図

作図ツールは `mathcha` のような体験を参考にする。

要確認:

- `Mathcha` に近い UI を目指すのか。
- 画像としての図を作るのか、TeX/TikZ などのコードを生成するのか。
- プレビュー側と UI 側のどちらに主導権を置くのか。

## テンプレート

カスタムテンプレートを作成できるようにする。

要確認:

- 「ガチカスタムテンプレート」の意味でよいか。
- テンプレートは `.tex` / `.sty` / UI 設定のどこまで含めるか。

## 実装メモ

- TeX コードを隠す編集レンズを広げていく。
- 単純なコマンド引数だけでなく、参照、章節、図表、目次なども UI オブジェクトとして扱う。
- 複雑なケースは最初から完全に隠そうとせず、必要に応じてコード編集へ逃がせる設計にする。
- プレビューは最終 PDF に近い見た目を維持する。
- AI とのやり取りで生成された内容も、最終的には TeX DOM / 文書構造へ反映する。

## 未確定・要確認

- 「AI を使った資料版」の正確な意味。
- ネイティブアプリ化後に追加する `minimum TeX` 支援の操作単位。
- 作図ツールの保存形式。
- カスタムテンプレートの範囲。
- 有料機能として扱う画面・機能の境界。
